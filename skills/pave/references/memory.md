# Memory

## Purpose

Capture durable repo knowledge without mixing it with active task
state.

## Capture When Useful

- A product, architecture, design, or deployment decision was made.
- A bug root cause was found.
- Verification commands or setup rules were clarified.
- A recurring failure mode was fixed.

## Rules

Prefer repo-local `.wiki/` when present. Capture concise verified
facts, not chat history. Do not store secrets, credentials, or
sensitive data.
